---
title: Multi-Project & Shared Access
---

 <p>Local SEO is built to handle more than one business and more than one person. You can manage several businesses side by side, and you can invite teammates to help with any of them. This guide explains how that works so everyone on your team sees the right data and has the right level of access.</p>
<h2 id="locations-live-inside-projects">Locations live inside projects<a aria-label="Copy link to Locations live inside projects" class="heading-anchor" href="#locations-live-inside-projects">#</a></h2>
<p>Every business location you add to Local SEO belongs to a <strong>project</strong>. A project is the home for one business and everything about it: its locations, Google Business Profile, posts, reviews, and rankings.</p>
<ul><li>A single project can hold one or more locations (for example, a business with several branches).</li><li>If you manage more than one business, each one is its own project. Switch between them using the project selector at the top of the dashboard.</li><li>A project can have multiple team members, so you and your colleagues can work on the same business together.</li></ul>
<p>When you first set up Local SEO for a business, theStacc either adds it to an existing project or creates a brand-new project for it. See <a href="/docs/local-seo/google-business-profile/">Google Business Profile</a> for the full onboarding walkthrough.</p>
<h2 id="who-can-do-what">Who can do what<a aria-label="Copy link to Who can do what" class="heading-anchor" href="#who-can-do-what">#</a></h2>
<p>Team members get access to a project in one of two ways, and both lead to a clear set of permissions:</p>
<ul><li><strong>Members of the account that owns the project</strong> have full access to it.</li><li><strong>Invited members</strong> are added to a specific project with a role of <strong>Admin</strong>, <strong>Editor</strong>, or <strong>Viewer</strong>.</li></ul>
<p>What each level can do in Local SEO:</p>
<ul><li><strong>Viewer</strong> - can see the dashboard, locations, posts, reviews, and rankings, but cannot make changes.</li><li><strong>Editor (or better)</strong> - can make changes: add and remove locations, edit and generate posts, complete onboarding for a project, and start Local SEO onboarding on an existing project.</li><li><strong>Admin</strong> - can do everything an Editor can, plus create new projects and start a Local SEO trial.</li></ul>
<p>In short: editing is open to Editors and Admins, while the two actions that affect billing or create new workspaces - <strong>starting a trial</strong> and <strong>creating a new project</strong> - are reserved for Admins.</p>
<p>You invite people and set their role under <strong>Settings &gt; Members</strong>. For the full picture of roles, invitations, and share links, see <a href="/docs/account/workspace-team/">Workspace &amp; Team</a>.</p>
<h2 id="invited-members-always-see-the-right-data">Invited members always see the right data<a aria-label="Copy link to Invited members always see the right data" class="heading-anchor" href="#invited-members-always-see-the-right-data">#</a></h2>
<p>When you open a project, Local SEO uses the *project's* owning account to decide which subscription, trial, location limit, and data to show - not the account of whoever happens to be logged in.</p>
<p>This matters most for shared projects. If a colleague invites you to help manage their business, you will see that business's real trial status, its location limit, and its locations and posts - exactly as the owner sees them. You do not need your own separate subscription to view or work on a project you have been invited to. The owner's plan is what counts.</p>
<h3 id="the-account-level-fallback">The account-level fallback<a aria-label="Copy link to The account-level fallback" class="heading-anchor" href="#the-account-level-fallback">#</a></h3>
<p>Most setups today are project-based: the subscription or trial is attached to a specific project. Some older accounts were set up at the account level instead, with no specific project attached.</p>
<p>theStacc handles both automatically. When it checks your trial or subscription for a project, it first looks for a plan attached to that project. If it does not find one, it falls back to an account-wide plan. This means older account-level customers keep working without interruption, and newer project-level customers get billing that is scoped to each business.</p>
<h2 id="how-a-project-becomes-a-local-seo-project">How a project becomes a Local SEO project<a aria-label="Copy link to How a project becomes a Local SEO project" class="heading-anchor" href="#how-a-project-becomes-a-local-seo-project">#</a></h2>
<p>A project only shows the Local SEO section in the sidebar once it actually has a location. Here is the lifecycle:</p>
<ol><li><strong>You start onboarding.</strong> When you begin setting up Local SEO for a business, the project is marked as *started*. The Local SEO section does not appear yet - this prevents an empty, half-finished section from showing up if you stop partway through.</li><li><strong>You add your first location.</strong> When you confirm your business and create its first location, Local SEO turns on for that project. The Local SEO section now appears in the sidebar, and you can manage that business.</li><li><strong>You finish onboarding.</strong> Once setup is complete, the project's Local SEO status moves from *started* to *completed*, and theStacc begins generating your initial posts.</li></ol>
<p>If you ever remove the last remaining location from a project, Local SEO turns back off for that project and the section is hidden again - so the menu never links to empty pages. Adding a new location later turns it back on.</p>
<h2 id="trials-and-projects">Trials and projects<a aria-label="Copy link to Trials and projects" class="heading-anchor" href="#trials-and-projects">#</a></h2>
<p>A Local SEO trial runs for 3 days. When you start a trial on a project, it is tied to that project, so a teammate you have invited will see the same trial status you do.</p>
<p>Only an Admin can start a trial, because it is a change to the account's plan. After the trial, continuing with Local SEO uses your subscription. For pricing details, see the <a href="/docs/account/billing/">Billing</a> guide.</p>
<h2 id="putting-it-together">Putting it together<a aria-label="Copy link to Putting it together" class="heading-anchor" href="#putting-it-together">#</a></h2>
<ul><li>Each business is a <strong>project</strong>; each project can hold one or more locations and one or more team members.</li><li><strong>Editors and Admins</strong> can make changes; only <strong>Admins</strong> can create new projects or start a trial.</li><li>Whichever project you are working in, theStacc uses <strong>that project's account</strong> for its plan, limits, and data - so invited members always see the right business.</li><li><strong>Older account-level setups</strong> keep working through an automatic fallback.</li></ul>
<p>For setting up a business from scratch, see <a href="/docs/content-seo/business-setup/">Business Setup</a> and <a href="/docs/local-seo/google-business-profile/">Google Business Profile</a>. To invite teammates and manage roles, see <a href="/docs/account/workspace-team/">Workspace &amp; Team</a>.</p>
