---
title: Platform Character Limits
---

 <p>Every social network caps how long a post caption can be, and the caps are very different from one platform to the next. theStacc knows each platform's limit and checks your caption against it before anything goes live, so you never get a half-published post where one network accepted it and another rejected it.</p>
<p>This page lists the exact limits, explains the quirky way X (Twitter) counts characters, and shows how the editor keeps you informed while you write.</p>
<h2 id="the-hard-limits">The hard limits<a aria-label="Copy link to The hard limits" class="heading-anchor" href="#the-hard-limits">#</a></h2>
<p>These are the maximum caption lengths theStacc enforces per platform:</p>
<div class="table-wrap"><table><thead><tr><th>Platform</th><th>Character limit</th></tr></thead><tbody><tr><td><strong>X (Twitter)</strong></td><td>280 (free) / 25,000 (Premium)</td></tr><tr><td><strong>Bluesky</strong></td><td>300</td></tr><tr><td><strong>Threads</strong></td><td>500</td></tr><tr><td><strong>Instagram</strong></td><td>2,200</td></tr><tr><td><strong>LinkedIn</strong></td><td>3,000</td></tr><tr><td><strong>Facebook</strong></td><td>63,206</td></tr></tbody></table></div>
<p>Facebook's limit is so high (around 63,000 characters) that you'll effectively never hit it with a normal caption. The tight ones to watch are <strong>X</strong>, <strong>Bluesky</strong>, and <strong>Threads</strong> - those are where most over-limit warnings show up.</p>
<p>theStacc also tracks limits for a few other platforms it can publish to: <strong>TikTok</strong> (2,200), <strong>Pinterest</strong> (500), and <strong>YouTube</strong> (5,000).</p>
<h3 id="x-premium-raises-the-limit-to-25-000">X Premium raises the limit to 25,000<a aria-label="Copy link to X Premium raises the limit to 25,000" class="heading-anchor" href="#x-premium-raises-the-limit-to-25-000">#</a></h3>
<p>The 280-character limit is X's free tier. If you pay for X Premium, your per-post ceiling jumps to 25,000 characters.</p>
<p>theStacc can't automatically tell whether your connected X account is on Premium, so by default it assumes the 280 limit. If you're a Premium subscriber, mark your X account as Premium under <strong>Social Media &gt; Settings &gt; Connections</strong> so the character counter uses the 25,000 limit instead of warning you at 280.</p>
<p>If you don't set this, a Premium user will see a false over-limit warning past 280 characters - the post would still publish fine on X, but the editor and pre-flight check play it safe with the lower number until you tell them otherwise.</p>
<h2 id="how-x-counts-characters-differently">How X counts characters differently<a aria-label="Copy link to How X counts characters differently" class="heading-anchor" href="#how-x-counts-characters-differently">#</a></h2>
<p>X doesn't simply count the letters in your caption. It uses its own rules, and theStacc mirrors them exactly so the count you see matches what X will actually measure. Two rules matter:</p>
<h3 id="links-always-count-as-23-characters">Links always count as 23 characters<a aria-label="Copy link to Links always count as 23 characters" class="heading-anchor" href="#links-always-count-as-23-characters">#</a></h3>
<p>X automatically shortens every link through its <code>t.co</code> wrapper, so it charges the same 23 characters for every URL no matter how long the actual link is. theStacc counts the same way:</p>
<ul><li>A full link like <code>https://example.com/a/very/long/path?with=params</code> counts as <strong>23 characters</strong>, not its real length.</li><li>A bare domain with no <code>https://</code> or <code>www.</code> - like <code>yourbusiness.com</code> or <code>example.io/page</code> - also counts as <strong>23 characters</strong>, because X detects and wraps these too.</li></ul>
<p>This bare-domain rule prevents a real trap: a caption that looks like it's exactly at 280 in a plain character count can actually be measured by X as over the limit, because a short bare domain like <code>yourbusiness.com</code> (16 characters) is weighed as 23. theStacc accounts for this so its count agrees with X's.</p>
<h3 id="emoji-and-non-latin-characters-count-as-2">Emoji and non-Latin characters count as 2<a aria-label="Copy link to Emoji and non-Latin characters count as 2" class="heading-anchor" href="#emoji-and-non-latin-characters-count-as-2">#</a></h3>
<p>X counts any character outside the basic Latin set as <strong>2 characters</strong>. That includes:</p>
<ul><li>Emoji</li><li>Chinese, Japanese, and Korean (CJK) text</li><li>Indic scripts like Devanagari and Tamil</li><li>Accented Latin letters</li></ul>
<p>So an emoji-heavy or non-English caption fills the 280 budget about twice as fast as plain English text. theStacc applies this same doubling, which is why the editor's counter for X can climb faster than the number of characters you typed.</p>
<p>These X-specific rules apply <strong>only to X</strong>. For every other platform, theStacc counts the caption's plain length - links and emoji are counted normally, character for character.</p>
<h2 id="the-pre-flight-check-blocks-an-over-limit-publish">The pre-flight check blocks an over-limit publish<a aria-label="Copy link to The pre-flight check blocks an over-limit publish" class="heading-anchor" href="#the-pre-flight-check-blocks-an-over-limit-publish">#</a></h2>
<p>When you publish or schedule a post, theStacc runs a pre-flight check before sending anything to the social networks. It looks at the exact caption that will go to each selected platform - including any per-platform caption override - and measures it against that platform's limit.</p>
<p>If any caption is too long, the publish is stopped with a <strong>caption too long</strong> error that names the offending platform and shows how far over you are (for example, <code>x (286/280)</code>). Nothing is sent to any platform.</p>
<p>This matters because of how social publishing works: without the check, X might reject an over-limit caption *after* theStacc had already published the same post to LinkedIn and Facebook. You'd be left with a post that's live on some networks and failed on others, which you'd then have to reconcile by hand. The pre-flight check catches the problem up front so this never happens.</p>
<p>When the check stops a publish, the fix is simple: shorten the caption for the platform that's over the limit, then publish again.</p>
<h2 id="the-live-counter-in-the-editor">The live counter in the editor<a aria-label="Copy link to The live counter in the editor" class="heading-anchor" href="#the-live-counter-in-the-editor">#</a></h2>
<p>You don't have to wait until publish time to find out a caption is too long. The post editor shows a live character counter that uses the <strong>same</strong> limits and the same X-specific counting rules as the pre-flight check.</p>
<p>To see it:</p>
<ol><li>Open any post to bring up the editor.</li><li>Click into the caption to start editing.</li><li>The counter sits above the caption, showing your current count over the platform's limit - for example <code>258 / 280</code>.</li></ol>
<p>The counter follows whichever platform tab you're on. If you've written a per-platform caption override for X, switching to the X tab shows that caption measured against X's rules; the <strong>main</strong> caption tab is measured against the platform you're previewing. As you type past the limit, the counter turns <strong>red</strong> so you can see at a glance that you've gone over.</p>
<h3 id="the-editor-warns-but-still-lets-you-save">The editor warns but still lets you save<a aria-label="Copy link to The editor warns but still lets you save" class="heading-anchor" href="#the-editor-warns-but-still-lets-you-save">#</a></h3>
<p>Going over the limit does <strong>not</strong> stop you from saving. theStacc treats an over-limit caption as a publish-time problem, not a save-time problem - people routinely write a long draft, save it, and trim it down later.</p>
<p>So when your caption is over the limit:</p>
<ul><li>The counter turns red.</li><li>The <strong>Save</strong> button stays enabled, but its label changes to <strong>Save (over limit)</strong> as a reminder that you'll need to trim before this can publish.</li><li>Your work is saved exactly as written.</li></ul>
<p>You fix the length on your own time, locally in the editor. Publishing is gated separately - the post simply won't publish until every selected platform's caption fits. If you try to publish anyway, the pre-flight check drops you straight into edit mode on the platform tab that's over, so you land right where you need to make the cut.</p>
<h2 id="tips-for-staying-within-limits">Tips for staying within limits<a aria-label="Copy link to Tips for staying within limits" class="heading-anchor" href="#tips-for-staying-within-limits">#</a></h2>
<ul><li><strong>Write per-platform captions for the short networks.</strong> A single caption that fits LinkedIn's 3,000 characters won't fit X's 280. Use a tighter override for X, Bluesky, and Threads. See <a href="/docs/social-media/content-scheduling/">Content &amp; Scheduling</a> for how to set per-platform captions.</li><li><strong>Remember links and emoji cost more on X.</strong> Every link is 23 characters and every emoji is 2 - budget for them.</li><li><strong>Watch the counter's color.</strong> Black means you're fine; red means trim before you publish.</li><li><strong>Mark your X account as Premium</strong> if you have it, so you get the full 25,000-character budget instead of the 280 default. You set this under <a href="/docs/social-media/connecting-platforms/">Connecting Platforms</a>.</li></ul>
