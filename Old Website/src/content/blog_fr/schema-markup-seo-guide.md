## Qu'est-ce que le schema markup

Le schema markup est du code que vous ajoutez à vos pages web pour aider les moteurs de recherche à comprendre le sens de votre contenu. Il traduit l'information lisible par les humains en un format structuré que les machines peuvent traiter.

Votre site web dit aux visiteurs : « Nous sommes ouverts du lundi au vendredi, de 8 h à 18 h. » Le schema markup dit la même chose à Google dans un format qu'il peut afficher directement dans les résultats de recherche.

Le schema markup utilise un vocabulaire maintenu par [Schema.org](https://schema.org/), une collaboration entre Google, Bing, Yahoo et Yandex. Ce vocabulaire définit des centaines de types de contenu et de propriétés. Google recommande le JSON-LD (JavaScript Object Notation for Linked Data) comme format privilégié.

### Comment fonctionne le schema markup

Quand vous ajoutez du schema markup à une page, vous créez une « étiquette » de données structurées pour votre contenu. Google lit cette étiquette, la valide et affiche l'information sous forme de rich result dans la recherche.

**Sans schema :** Google voit du texte sur une page et fait de son mieux pour en deviner le sens.

**Avec schema :** Google sait exactement que « 4,8 » est une note d'avis, que « 127 » est un nombre d'avis et que « lun-ven 8 h-18 h » sont vos horaires d'ouverture.

![Comparaison des résultats de recherche : sans schema vs avec schema markup](/images/blog/schema-with-vs-without.webp)

### Schema markup, rich results et rich snippets

Ces termes sont souvent confondus.

- **Schema markup** est le code que vous ajoutez à votre page
- **Données structurées** est le concept plus large d'organisation de l'information pour les machines
- **Rich results** sont les résultats de recherche enrichis que Google affiche grâce à vos données structurées
- **Rich snippets** est un terme plus ancien désignant le même concept

Vous ajoutez du schema markup. Google décide d'afficher ou non un rich result. Le schema ne garantit pas un rich result, mais sans schema, les rich results sont impossibles pour la plupart des types de contenu.

En 2026, le schema markup a un impact au-delà des résultats de recherche traditionnels de Google. Les systèmes d'IA comme les AI Overviews de Google, ChatGPT et Perplexity s'appuient sur les données structurées pour comprendre, résumer et citer votre contenu avec précision.

Les sites dotés d'un schema correct sont cités par les outils de recherche par IA bien plus souvent. Les données structurées rendent votre contenu lisible par les machines à un niveau granulaire. Le schema markup élimine l'ambiguïté. Le mot « apple » peut désigner un fruit, une entreprise technologique ou un label de musique. Le schema indique à Google lequel vous visez exactement. Cette précision aide Google à faire correspondre votre contenu aux bonnes requêtes de recherche.

Le schema markup alimente directement les Knowledge Panels et les featured snippets de Google. Ces affichages enrichis occupent plus d'espace visuel dans les résultats de recherche. Plus d'espace signifie plus d'attention. Plus d'attention signifie plus de clics. ## Les 10 types de schema les plus importants

Tous les types de schema n'ont pas le même poids. Concentrez-vous sur ceux qui déclenchent des rich results et correspondent à votre contenu.

![Types de schema markup essentiels pour le SEO avec niveaux de priorité](/images/blog/schema-types-table.webp)

### Schema Organization

Chaque site web a besoin du schema Organization. Il définit le nom de votre entreprise, le logo, l'URL du site web, les profils de réseaux sociaux et les coordonnées. Google utilise ces données pour les Knowledge Panels.

**Propriétés à inclure :** name, url, logo, sameAs (profils sociaux), contactPoint, address, description.

### Schema LocalBusiness

Pour toute entreprise ayant un lieu physique, le schema LocalBusiness est essentiel. Il fournit à Google votre adresse, votre numéro de téléphone, vos horaires, votre gamme de prix et votre zone de service. Ces données alimentent les résultats du pack local et la recherche locale.

Pour un guide complet sur l'optimisation du référencement local, consultez notre [guide Google Business Profile](/blog_fr/optimize-google-business-profile/).

### Schema Article et BlogPosting

Chaque article de blog et chaque publication doit inclure le schema Article ou BlogPosting. Il indique à Google le titre, l'auteur, la date de publication, la date de modification et l'image à la une. Ces données permettent d'obtenir des rich results de type article avec les dates et les informations sur l'auteur.

Pour la mise en œuvre du schema spécifique aux blogs, consultez notre [guide schema markup pour les articles de blog](/blog/schema-markup-for-blog-posts/).

### Schema Product

Les pages e-commerce ont besoin du schema Product avec le prix, la disponibilité, la devise, la marque et les données d'avis. Les rich results de produit affichent les prix et l'état du stock directement dans la recherche. Ce sont parmi les rich results les plus convertissants.

### Schema FAQPage

Le schema FAQPage balise des paires de questions-réponses. Google peut afficher des sections FAQ dépliables directement dans les résultats de recherche. Note : depuis 2024, Google limite les rich results FAQ aux sites gouvernementaux et de santé faisant autorité pour la plupart des requêtes. Mais ce schema aide toujours les systèmes d'IA à comprendre votre contenu FAQ.

### Schema HowTo

Le schema HowTo balise des instructions étape par étape. Google peut afficher des étapes numérotées, des estimations de temps et les outils nécessaires dans les résultats de recherche. Cela fonctionne bien pour les tutoriels et les guides.

### Schema BreadcrumbList

Le schema BreadcrumbList affiche la hiérarchie de navigation de votre site dans les résultats de recherche. Au lieu d'afficher une URL simple comme `example.com/blog/schema-markup-seo-guide`, Google affiche un fil d'Ariane cliquable : Accueil > Blog > Conseils SEO > Guide Schema Markup.

BreadcrumbList est l'un des types de schema les plus faciles à mettre en œuvre sur l'ensemble du site. Ajoutez-le une fois à votre modèle de page et il s'applique automatiquement à chaque page. Le bénéfice SEO est universel pour tous les types de contenu.

Pour une architecture de site et un [maillage interne](/blog/internal-linking-blog-posts/) corrects, les fils d'Ariane renforcent la structure hiérarchique que Google utilise pour comprendre l'organisation de votre contenu.

### Schema Review et AggregateRating

Le schema Review affiche des étoiles dans les résultats de recherche. AggregateRating montre la note moyenne et le nombre total d'avis. Les étoiles dans les résultats de recherche sont parmi les leviers de CTR les plus puissants. Suivez notre [guide pour obtenir plus d'avis Google](/blog/get-more-google-reviews-local-business/) pour développer le volume d'avis qui rend ce schema précieux.

### Schema Event

Le schema Event balise des événements avec les dates, les lieux, les prix des billets et les artistes. Google affiche des cartes d'événement avec les dates et les lieux directement dans les résultats de recherche. Indispensable pour les entreprises qui organisent des événements, des ateliers, des webinaires ou des conférences.

Incluez le nom de l'événement, la date de début, la date de fin, le lieu (avec adresse), l'URL des billets et la fourchette de prix. Google utilise ces données pour afficher vos événements dans les résultats de recherche dédiés et les listes d'événements Google Maps.

### Schema VideoObject

Le schema VideoObject balise les contenus vidéo avec le titre, la description, la miniature, la durée et la date de mise en ligne. Google peut afficher des miniatures vidéo dans les résultats de recherche. Les rich results vidéo se démarquent visuellement et obtiennent des taux de clic nettement supérieurs aux résultats textuels.

Incluez le nom de la vidéo, la description, l'URL de la miniature, la date de mise en ligne, la durée et l'URL du contenu. Si votre contenu inclut des vidéos YouTube ou Vimeo intégrées, le schema VideoObject aide Google à les faire apparaître dans les résultats de recherche vidéo.

## Comment ajouter du schema markup à votre site web

Ajouter du schema markup est plus simple que la plupart des gens ne le pensent. Le JSON-LD est le format recommandé. Il se place dans la section `<head>` de votre HTML sans modifier le contenu visible.

![5 étapes pour mettre en œuvre le schema markup sur votre site](/images/blog/schema-implementation-steps.webp)

### Étape 1 : choisir votre type de schema

Commencez par les types les plus pertinents pour votre activité :

| Type d'activité | Commencez par ces types de schema |
|---|---|
| Entreprise locale | Organization + LocalBusiness + BreadcrumbList |
| E-commerce | Organization + Product + BreadcrumbList + Review |
| Blog ou éditeur | Organization + Article/BlogPosting + BreadcrumbList |
| Entreprise de services | Organization + LocalBusiness + FAQPage + Service |
| Restaurant | Organization + Restaurant + Menu + BreadcrumbList |

### Étape 2 : générer le code JSON-LD

Utilisez un outil de génération de schema pour créer le code. Plusieurs options gratuites existent :

- [Google Structured Data Markup Helper](https://www.google.com/webmasters/markup-helper/)
- [TechnicalSEO.com Schema Generator](https://technicalseo.com/tools/schema-markup-generator/)
- [Schemantra JSON-LD Generator](https://schemantra.com/)

Saisissez les informations de votre entreprise. L'outil génère le bloc de code JSON-LD.

![Exemple de code schema markup JSON-LD pour une entreprise locale](/images/blog/schema-json-ld-example.webp)

### Étape 3 : ajouter le code à votre site web

Collez le bloc de code JSON-LD dans la section `<head>` de votre page HTML, à l'intérieur d'une balise `<script type="application/ld+json">`.

**Pour WordPress :** utilisez une extension comme Rank Math, Yoast SEO ou Schema Pro. Celles-ci ajoutent le schema automatiquement sans éditer de code. Rank Math gère la plupart des types de schema nativement et génère automatiquement le schema Article pour chaque publication.

**Pour Astro, Next.js ou les sites statiques :** ajoutez le script JSON-LD directement à votre modèle de page ou à votre composant head. Créez un composant de schema réutilisable qui accepte des props pour les données spécifiques. Cela maintient votre schema cohérent et facile à mettre à jour sur des centaines de pages.

**Pour Shopify :** utilisez les données structurées intégrées ou une application schema du Shopify App Store. La plupart des thèmes Shopify incluent un schema Product de base. Des applications comme JSON-LD for SEO ajoutent automatiquement les schemas Organization, BreadcrumbList et FAQPage.

**Pour Webflow :** ajoutez du code personnalisé dans la section head des paramètres de la page. Webflow n'a pas de support natif du schema, vous devez donc ajouter le JSON-LD manuellement ou utiliser une intégration tierce.

### Étape 4 : valider votre balisage

Après avoir ajouté le schema, validez-le avec 2 outils :

- **[Google Rich Results Test](https://search.google.com/test/rich-results) :** vérifie si votre schema est éligible aux rich results
- **[Schema.org Validator](https://validator.schema.org/) :** valide la correction technique de votre balisage

Corrigez les erreurs ou avertissements avant de passer à la suite. Les problèmes courants incluent des propriétés obligatoires manquantes, des types de données incorrects et des valeurs incohérentes.

### Étape 5 : surveiller dans Google Search Console

Google Search Console rapporte les erreurs de schema et les performances des rich results. Consultez la section « Améliorations » pour identifier les problèmes. Surveillez quelles pages obtiennent des rich results et lesquelles présentent des erreurs.

Pour un guide complet de configuration et de surveillance de Search Console, consultez notre [guide Google Search Console](/blog/google-search-console-guide/).

## Schema markup et recherche par IA

En 2026, le schema markup fait plus qu'obtenir des rich results dans Google. Il rend votre contenu lisible par les systèmes d'IA qui propulsent la prochaine génération de recherche.

Les AI Overviews de Google, la recherche web de ChatGPT, Perplexity et Gemini extraient tous des données structurées des sites web. Lorsque votre page dispose d'un schema markup clair, les modèles d'IA peuvent extraire des faits avec certitude au lieu d'interpréter du texte non structuré.

### Comment le schema aide l'IA à citer votre contenu

Les systèmes d'IA privilégient les sources disposant de données claires et lisibles par les machines. Le schema markup fournit :

- **Clarté des entités.** L'IA connaît le nom de votre entreprise, son type, son emplacement et ses offres sans deviner.
- **Extraction des faits.** Les prix, les notes, les horaires et les spécifications sont structurés comme des données, pas enfouis dans des paragraphes.
- **Attribution des sources.** Lorsque l'IA cite une affirmation spécifique, les données structurées font de votre page la source évidente.

Les sites dotés d'un schema correct sont cités dans les réponses générées par IA 67 % plus souvent selon des recherches récentes. Pour en savoir plus sur l'optimisation pour la recherche par IA, consultez notre [guide pour être cité dans la recherche par IA](/blog/get-cited-ai-search/).

### Les types de schema les plus importants pour l'IA

- Les schemas **Organization** et **Person** aident l'IA à identifier qui vous êtes
- Le schema **Article** aide l'IA à comprendre le contexte de publication et l'auteur
- Le schema **FAQPage** fournit des paires de questions-réponses préstructurées que les modèles d'IA peuvent utiliser directement
- Le schema **HowTo** donne à l'IA des instructions étape par étape structurées à référencer

Consultez notre [guide d'optimisation pour les AI Overviews de Google](/blog/optimize-google-ai-overviews/) pour la stratégie complète de recherche par IA.

## Erreurs courantes de schema markup

La plupart des mises en œuvre de schema comportent des erreurs qui empêchent l'affichage des rich results. Évitez-les.

**Propriétés obligatoires manquantes.** Chaque type de schema a des propriétés obligatoires et recommandées. Omettre les propriétés obligatoires provoque des échecs de validation. Vérifiez toujours la [documentation sur les données structurées de Google](https://developers.google.com/search/docs/appearance/structured-data/intro-structured-data) pour connaître les exigences.

**Balisage de contenu masqué.** Le schema doit décrire du contenu visible sur la page. N'ajoutez pas de schema pour du contenu situé dans des divs masqués, des pop-ups ou des éléments que les utilisateurs ne peuvent pas voir. Google considère cela comme trompeur.

**Balisage d'avis auto-promotionnel.** Vous ne pouvez pas ajouter de schema Review à votre propre produit sur votre propre site web sur la base de votre propre évaluation. Les avis doivent provenir de tiers. Google pénalise le balisage d'avis autoréférentiel.

**Types de données incorrects.** Un prix doit être un nombre, pas un texte comme « Contactez-nous pour les tarifs. » Une date doit respecter le format ISO 8601 (AAAA-MM-JJ). Les types incorrects provoquent des échecs silencieux.

**Schema en double ou contradictoire.** Plusieurs blocs de schema avec des informations contradictoires confusent Google. Vos schemas Organization et LocalBusiness ne doivent pas afficher des adresses ou des numéros de téléphone différents.

**Ne pas mettre à jour le schema quand le contenu change.** Si vos horaires, vos tarifs ou vos services changent, mettez à jour votre schema en conséquence. Un schema obsolète induit en erreur les utilisateurs et Google. Effectuez des [audits SEO](/blog_fr/how-to-do-seo-audit/) réguliers incluant la validation du schema.

## FAQ

**Le schema markup améliore-t-il directement le classement Google ?**

Non. Google a confirmé que le schema markup n'est pas un facteur de classement direct. Il n'améliore pas votre position dans les résultats de recherche à lui seul. Mais les rich results qu'il permet d'obtenir augmentent le taux de clic jusqu'à 40 %. Un CTR plus élevé génère plus de trafic, ce qui soutient indirectement les améliorations de classement au fil du temps.

**Quel est le meilleur format pour le schema markup ?**

Google recommande le JSON-LD. C'est le plus facile à mettre en œuvre, cela n'affecte pas la vitesse de chargement de la page et cela sépare les données structurées du contenu HTML. Microdata et RDFa sont des alternatives plus anciennes qui fonctionnent toujours mais sont plus difficiles à maintenir.

**Comment vérifier que mon schema markup fonctionne ?**

Utilisez le Google Rich Results Test (search.google.com/test/rich-results) pour valider votre balisage. Saisissez l'URL de votre page ou collez votre code. L'outil indique si votre schema est éligible aux rich results et signale les erreurs éventuelles. Consultez également la section « Améliorations » de Google Search Console pour une surveillance continue.

**Puis-je ajouter du schema markup sans coder ?**

Oui. Les extensions WordPress comme Rank Math, Yoast SEO et Schema Pro ajoutent le schema automatiquement. Les applications Shopify gèrent le schema pour l'e-commerce. Les constructeurs de sites comme Wix et Squarespace incluent un schema de base dans leurs modèles. Pour les implémentations personnalisées, des outils de génération gratuits créent le code que vous n'avez plus qu'à coller.

**Quels types de schema offrent le plus grand bénéfice SEO ?**

Les schemas Organization, LocalBusiness et Article sont les types les plus prioritaires. Le schema Product avec les prix et les avis offre les meilleurs résultats e-commerce. Le schema BreadcrumbList a un bénéfice quasi universel pour tout site. Concentrez-vous sur ceux-là avant d'ajouter des types plus spécialisés.

**Stacc ajoute-t-il du schema markup aux articles de blog ?**

Chaque article de blog publié par Stacc inclut les schemas Article et BlogPosting corrects avec les informations sur l'auteur, la date de publication et le balisage de l'image à la une. Nos articles suivent les [meilleures pratiques de SEO on-page](/blog_fr/on-page-seo-guide/) incluant les données structurées, le maillage interne et l'optimisation des mots-clés.
