## O Que É Schema Markup

Schema markup é um código que você adiciona às suas páginas web para ajudar os mecanismos de busca a entender o significado do seu conteúdo. Ele traduz informações legíveis para humanos em um formato estruturado que as máquinas podem processar.

O seu site diz aos visitantes "Estamos abertos de segunda a sexta, das 8h às 18h." O schema markup diz o mesmo ao Google em um formato que ele pode exibir diretamente nos resultados de busca.

O schema markup usa um vocabulário mantido pelo [Schema.org](https://schema.org/), uma colaboração entre Google, Bing, Yahoo e Yandex. O vocabulário define centenas de tipos de conteúdo e propriedades. O Google recomenda o JSON-LD (JavaScript Object Notation for Linked Data) como o formato preferido.

### Como o Schema Markup Funciona

Quando você adiciona schema markup a uma página, você cria um "rótulo" de dados estruturados para o seu conteúdo. O Google lê esse rótulo, valida-o e exibe a informação como um rich result na busca.

**Sem schema:** O Google vê texto em uma página e faz a sua melhor suposição sobre o que ele significa.

**Com schema:** O Google sabe exatamente que "4,8" é uma avaliação, "127" é uma contagem de avaliações e "Seg-Sex 8h-18h" é o seu horário de funcionamento.

![Comparação de resultados de busca: sem schema vs com schema markup](/images/blog/schema-with-vs-without.webp)

### Schema Markup vs Rich Results vs Rich Snippets

Esses termos são confundidos constantemente.

- **Schema markup** é o código que você adiciona à sua página
- **Dados estruturados** é o conceito mais amplo de organizar informações para máquinas
- **Rich results** são os resultados de busca aprimorados que o Google exibe usando seus dados estruturados
- **Rich snippets** é um termo mais antigo para o mesmo conceito

Você adiciona schema markup. O Google decide se exibe um rich result. O schema não garante um rich result, mas sem schema, rich results são impossíveis para a maioria dos tipos de conteúdo.

## Os 10 Tipos de Schema Mais Importantes

Nem todos os tipos de schema têm o mesmo peso. Foque nos que disparam rich results e combinam com o seu conteúdo.

![Tipos essenciais de schema markup para SEO com níveis de prioridade](/images/blog/schema-types-table.webp)

### Schema de Organização

Todo site precisa de schema de Organização. Ele define o nome da sua empresa, logotipo, URL do site, perfis de mídia social e informações de contato. O Google usa esses dados para Painéis de Conhecimento.

**Propriedades para incluir:** name, url, logo, sameAs (perfis sociais), contactPoint, address, description.

### Schema de LocalBusiness

Para qualquer empresa com local físico, o schema de LocalBusiness é essencial. Ele fornece ao Google seu endereço, telefone, horário, faixa de preço e área de atendimento. Esses dados alimentam os listados do pacote de mapas e os resultados de busca local.

Para um guia completo de otimização local, leia nosso [guia de Google Business Profile](/blog/optimize-google-business-profile).

### Schema de Article e BlogPosting

Todo post de blog e artigo deve incluir schema de Article ou BlogPosting. Ele diz ao Google o título, autor, data de publicação, data de modificação e imagem em destaque. Esses dados habilitam rich results no estilo de artigo com datas e informações do autor.

Para implementação de schema específica para blogs, leia nosso [guia de schema markup para posts de blog](/blog/schema-markup-for-blog-posts).

### Schema de Produto

Páginas de e-commerce precisam de schema de Produto com preço, disponibilidade, moeda, marca e dados de avaliação. Rich results de produto mostram preços e status de estoque diretamente na busca. Esses são entre os rich results com maior taxa de conversão.

### Schema de FAQPage

O schema de FAQPage marca pares de pergunta-resposta. O Google pode exibir seções de FAQ expansíveis diretamente nos resultados de busca. Nota: desde 2024, o Google restringe rich results de FAQ para sites governamentais e de saúde autoritativos na maioria das consultas. Mas o schema ainda ajuda sistemas de IA a entender seu conteúdo de FAQ.

### Schema de HowTo

O schema de HowTo marca instruções passo a passo. O Google pode exibir passos numerados, estimativas de tempo e ferramentas necessárias nos resultados de busca. Isso funciona bem para conteúdo de tutorial e guias.

### Schema de BreadcrumbList

O schema de BreadcrumbList exibe a hierarquia de navegação do seu site nos resultados de busca. Em vez de mostrar uma URL simples como `example.com/blog/schema-markup-seo-guide`, o Google mostra uma trilha clicável: Início > Blog > Dicas de SEO > Guia de Schema Markup.

BreadcrumbList é um dos tipos de schema mais fáceis de implementar em todo o site. Adicione-o ao seu template de página uma vez e ele se aplica a todas as páginas automaticamente. O benefício de SEO é universal em todos os tipos de conteúdo.

Para arquitetura de site adequada e [links internos](/blog/internal-linking-blog-posts), breadcrumbs reforçam a estrutura hierárquica que o Google usa para entender a organização do seu conteúdo.

### Schema de Review e AggregateRating

O schema de Review exibe avaliações em estrelas nos resultados de busca. O AggregateRating mostra a pontuação média e o total de avaliações. Avaliações em estrelas nos resultados de busca estão entre os impulsionadores de CTR mais poderosos. Siga nosso [guia para conseguir mais avaliações no Google](/blog/get-more-google-reviews-local-business) para construir o volume de avaliações que torna esse schema valioso.

### Schema de Evento

O schema de Evento marca eventos com datas, locais, preços de ingressos e artistas. O Google exibe cards de evento com datas e locais diretamente nos resultados de busca. Essencial para empresas que promovem eventos, workshops, webinars ou conferências.

Inclua o nome do evento, data de início, data de término, local (com endereço), URL de ingressos e faixa de preço. O Google usa esses dados para exibir seus eventos em resultados de busca dedicados a eventos e listagens de eventos no Google Maps.

### Schema de VideoObject

O schema de VideoObject marca conteúdo de vídeo com título, descrição, miniatura, duração e data de upload. O Google pode exibir miniaturas de vídeo nos resultados de busca. Rich results de vídeo se destacam visualmente e ganham taxas de clique significativamente mais altas do que listagens apenas de texto.

Inclua o nome do vídeo, descrição, URL da miniatura, data de upload, duração e URL do conteúdo. Se seu conteúdo inclui vídeos incorporados do YouTube ou Vimeo, o schema de VideoObject ajuda o Google a exibi-los nos resultados de busca de vídeo.

## Como Adicionar Schema Markup ao Seu Site

Adicionar schema markup é mais simples do que a maioria das pessoas pensa. JSON-LD é o formato recomendado. Ele vai na seção `<head>` do seu HTML sem alterar nenhum conteúdo visível.

![5 passos para implementar schema markup no seu site](/images/blog/schema-implementation-steps.webp)

### Passo 1: Escolha Seu Tipo de Schema

Comece com os tipos mais relevantes para o seu negócio:

| Tipo de Negócio | Comece Com Esses Tipos de Schema |
|---|---|
| Negócio local | Organização + LocalBusiness + BreadcrumbList |
| E-commerce | Organização + Produto + BreadcrumbList + Review |
| Blog ou editora | Organização + Article/BlogPosting + BreadcrumbList |
| Negócio de serviços | Organização + LocalBusiness + FAQPage + Service |
| Restaurante | Organização + Restaurant + Menu + BreadcrumbList |

### Passo 2: Gere o Código JSON-LD

Use uma ferramenta geradora de schema para criar o código. Existem várias opções gratuitas:

- [Google Structured Data Markup Helper](https://www.google.com/webmasters/markup-helper/)
- [TechnicalSEO.com Schema Generator](https://technicalseo.com/tools/schema-markup-generator/)
- [Schemantra JSON-LD Generator](https://schemantra.com/)

Insira as informações do seu negócio. A ferramenta gera o bloco de código JSON-LD.

![Exemplo de código de schema markup JSON-LD para um negócio local](/images/blog/schema-json-ld-example.webp)

### Passo 3: Adicione o Código ao Seu Site

Cole o bloco de código JSON-LD na seção `<head>` do HTML da sua página dentro de uma tag `<script type="application/ld+json">`.

**Para WordPress:** Use um plugin como Rank Math, Yoast SEO ou Schema Pro. Eles adicionam schema automaticamente sem editar código. O Rank Math lida com a maioria dos tipos de schema prontos para uso e gera schema de Article automaticamente para cada post.

**Para Astro, Next.js ou sites estáticos:** Adicione o script JSON-LD diretamente ao seu template de página ou componente de head. Crie um componente de schema reutilizável que aceite props para os dados específicos. Isso mantém seu schema consistente e fácil de atualizar em centenas de páginas.

**Para Shopify:** Use os dados estruturados integrados ou um app de schema da Shopify App Store. A maioria dos temas Shopify inclui schema básico de Produto. Apps como JSON-LD for SEO adicionam schema de Organização, BreadcrumbList e FAQPage automaticamente.

**Para Webflow:** Adicione código personalizado na seção de head das configurações da página. O Webflow não tem suporte nativo a schema, então você precisa adicionar JSON-LD manualmente ou usar uma integração de terceiros.

### Passo 4: Valide Seu Markup

Depois de adicionar o schema, valide-o usando 2 ferramentas:

- **[Google Rich Results Test](https://search.google.com/test/rich-results):** Verifica se seu schema qualifica para rich results
- **[Schema.org Validator](https://validator.schema.org/):** Valida a correção técnica do seu markup

Corrija quaisquer erros ou avisos antes de prosseguir. Problemas comuns incluem propriedades obrigatórias ausentes, tipos de dados incorretos e valores incompatíveis.

### Passo 5: Monitore no Google Search Console

O Google Search Console reporta erros de schema e desempenho de rich results. Verifique a seção "Aprimoramentos" para problemas. Monitore quais páginas ganham rich results e quais têm erros.

Para um guia completo de configuração e monitoramento do Search Console, leia nosso [guia do Google Search Console](/blog/google-search-console-guide).

## Schema Markup e Busca por IA

Em 2026, o schema markup faz mais do que conquistar rich results no Google. Ele torna seu conteúdo legível para sistemas de IA que impulsionam a próxima geração de busca.

Google AI Overviews, ChatGPT web search, Perplexity e Gemini todos extraem dados estruturados de sites. Quando sua página tem schema markup claro, modelos de IA podem extrair fatos com confiança em vez de interpretar texto não estruturado.

### Como o Schema Ajuda a IA a Citar Seu Conteúdo

Sistemas de IA preferem fontes com dados claros e legíveis para máquinas. O schema markup fornece:

- **Claridade de entidade.** A IA sabe o nome do seu negócio, tipo, local e ofertas sem supor.
- **Extração de fatos.** Preços, avaliações, horários e especificações são estruturados como dados, não enterrados em parágrafos.
- **Atribuição de fonte.** Quando a IA cita uma alegação específica, dados estruturados tornam sua página a fonte óbvia.

Sites com schema adequado são citados em respostas geradas por IA 67% mais vezes, de acordo com pesquisas recentes. Para mais sobre otimização para busca por IA, leia nosso [guia para ser citado em buscas por IA](/blog/get-cited-ai-search).

### Tipos de Schema Que Mais Importam para IA

- **Organization** e **Person** schema ajudam a IA a identificar quem você é
- **Article** schema ajuda a IA a entender contexto de publicação e autoria
- **FAQPage** schema fornece pares de Q&A pré-estruturados que modelos de IA podem usar diretamente
- **HowTo** schema dá à IA instruções passo a passo estruturadas para referenciar

Leia nosso [guia de otimização para AI Overviews](/blog/optimize-google-ai-overviews) para o playbook completo de busca por IA.

## Erros Comuns de Schema Markup

A maioria das implementações de schema tem erros que impedem rich results de aparecer. Evite estes.

**Propriedades obrigatórias ausentes.** Cada tipo de schema tem propriedades obrigatórias e recomendadas. Deixar de fora as obrigatórias causa falhas de validação. Sempre verifique a [documentação de dados estruturados do Google](https://developers.google.com/search/docs/appearance/structured-data/intro-structured-data) para requisitos.

**Marcar conteúdo oculto.** O schema deve descrever conteúdo visível na página. Não adicione schema para conteúdo em divs ocultos, popups ou elementos que os usuários não podem ver. O Google sinaliza isso como enganoso.

**Markup de avaliação autopromocional.** Você não pode adicionar schema de Review ao seu próprio produto no seu próprio site baseado na sua própria avaliação. As avaliações devem vir de terceiros. O Google penaliza markup de avaliação autorreferencial.

**Tipos de dados incorretos.** Um preço deve ser um número, não texto como "Entre em contato para preços." Uma data deve seguir o formato ISO 8601 (AAAA-MM-DD). Tipos incorretos causam falhas silenciosas.

**Schema duplicado ou conflitante.** Múltiplos blocos de schema com informações conflitantes confundem o Google. Seu schema de Organização e LocalBusiness não devem mostrar endereços ou telefones diferentes.

**Não atualizar o schema quando o conteúdo muda.** Se seu horário, preços ou serviços mudarem, atualize seu schema para corresponder. Schema desatizado engana usuários e o Google. Execute [auditorias de SEO](/blog/how-to-do-seo-audit) regulares que incluam validação de schema.

## FAQ

**O schema markup melhora diretamente os ranqueamentos no Google?**

Não. O Google confirmou que o schema markup não é um fator de ranqueamento direto. Ele não impulsiona sua posição nos resultados de busca por si só. Mas os rich results que ele habilita aumentam a taxa de cliques em até 40%. Um CTR mais alto gera mais tráfego, o que indiretamente apoia melhorias de ranqueamento ao longo do tempo.

**Qual é o melhor formato para schema markup?**

O Google recomenda JSON-LD. É o mais fácil de implementar, não afeta a velocidade de carregamento da página e separa os dados estruturados do seu conteúdo HTML. Microdata e RDFa são alternativas mais antigas que ainda funcionam, mas são mais difíceis de manter.

**Como verifico se meu schema markup está funcionando?**

Use o Google Rich Results Test (search.google.com/test/rich-results) para validar seu markup. Insira a URL da sua página ou cole seu código. A ferramenta mostra se seu schema qualifica para rich results e sinaliza quaisquer erros. Verifique também a seção "Aprimoramentos" do Google Search Console para monitoramento contínuo.

**Posso adicionar schema markup sem programar?**

Sim. Plugins do WordPress como Rank Math, Yoast SEO e Schema Pro adicionam schema automaticamente. Apps da Shopify lidam com schema para e-commerce. Construtores de sites como Wix e Squarespace incluem schema básico nos templates. Para implementações personalizadas, ferramentas geradoras gratuitas criam o código para você colar.

**Quais tipos de schema oferecem o maior benefício de SEO?**

Schemas de Organização, LocalBusiness e Article são os tipos de maior prioridade. O schema de Produto com preços e avaliações entrega os resultados de e-commerce mais fortes. O schema de BreadcrumbList tem benefício quase universal para qualquer site. Foque nesses antes de adicionar tipos mais especializados.

**A Stacc adiciona schema markup aos posts de blog?**

Todo post de blog publicado pela Stacc inclui schema adequado de Article e BlogPosting com informações do autor, data de publicação e markup de imagem em destaque. Nossos posts seguem [melhores práticas de SEO on-page](/blog/on-page-seo-guide) incluindo dados estruturados, links internos e otimização de palavras-chave.
