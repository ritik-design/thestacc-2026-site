# Keyword Optimization — Making the Page Win the SERP

## Step 1 — Identify the page's keyword set

Every page targets:
- **1 primary keyword** (matches the URL slug)
- **3–5 secondary keywords** (related variations, long-tails, PAA themes)
- **20+ tertiary terms** (entities, related phrases that get sprinkled naturally)

### How to derive them

For SEO-keyword landings the slug IS the primary. For other pages:

| Page type | Where to look |
|---|---|
| Module pages | The module's name + "software" / "tool" / "platform" |
| Blog post | The article topic phrased as a "how to X" / "what is X" / "best X" query |
| Glossary term | The term itself + "definition" + "meaning" |
| Reviews / alternatives / best-of | "{tool} review", "alternative to {tool}", "best {category}" |
| Case study | Customer name + vertical + outcome (e.g. "dental SEO results case study") |
| Vertical landing | "SEO for {vertical}" + "{vertical} marketing software" |

For research: Google's autocomplete + People Also Ask + "related searches" on the SERP for the primary keyword. Capture every variation that gets impressions. Add them to the page's secondary list.

## Step 2 — Place the primary keyword

Every primary keyword needs to land in these 8 places. Miss any one and ranking drops measurably.

| # | Location | Example |
|---|---|---|
| 1 | URL slug | `/local-seo-software/` |
| 2 | Meta title (in first 30 chars) | `Local SEO Software — Map Pack in 90 Days \| theStacc` |
| 3 | Meta description | "...the AI-powered local SEO software that puts you in the Map Pack..." |
| 4 | H1 | `Local SEO Software That Wins the Map Pack` |
| 5 | First 100 words of body | "Local SEO software should..." |
| 6 | At least 2 H2s | "How our local SEO software works", "Compare local SEO software" |
| 7 | Image alt text | `alt="Local SEO software dashboard showing Map Pack ranking"` |
| 8 | Internal anchor text from at least 3 other pages on the site | Other pages link `<a href="/local-seo-software/">local SEO software</a>` |

## Step 3 — Distribute secondary keywords

Each secondary keyword goes in:
- One H2 OR H3
- 2–3 mentions in body copy (natural, not stuffed)
- 1 image alt or caption
- 1 internal link's anchor text from elsewhere

## Step 4 — Build the entity stack

Beyond keywords, modern SEO ranks **entities** (people, places, concepts). Mention these on every relevant page:

- **Brand entities**: theStacc, Siddharth Gangal (founder)
- **Product entities**: Content SEO, Local SEO, Social Media
- **Competitor entities**: Semrush, Ahrefs, Surfer, Jasper, Frase, Clearscope (these signal "this page is about SEO software")
- **Platform entities**: Google, Google Business Profile, ChatGPT, Perplexity, Gemini, Claude, Copilot, WordPress, Webflow, Shopify, Ghost
- **Concept entities**: Map Pack, AI Overviews, GEO (Generative Engine Optimization), local SEO, schema markup, E-E-A-T

The more entity coverage on a page, the more "topical authority" Google assigns it.

## Step 5 — Match search intent

Classify the primary keyword's intent BEFORE writing:

| Intent | Signal | Page shape |
|---|---|---|
| **Informational** | "what is", "how does", "why" | Definition + explanation + examples + related concepts. CTA = newsletter or related tool. NOT pricing/trial. |
| **Navigational** | "{brand}", "{brand} login" | Direct, fast. One headline, one CTA. |
| **Commercial-investigation** | "best", "review", "alternative", "vs", "software", "tool" | Comparison-heavy. Tables. Multi-product mentions. CTA = $1 trial. |
| **Transactional** | "buy", "price", "pricing", "demo", "free trial" | Pricing visible above the fold. Trust signals. Multiple CTAs. |

Mismatched intent = wrong page shape = no ranking, no matter how well-written. Run the primary keyword through Google — what's ranking right now? Match that shape.

## Step 6 — Capture the SERP features

Modern SERPs are crowded. Pick which features to win:

| Feature | How to capture |
|---|---|
| **Featured snippet (paragraph)** | First 40–60 words after H1: direct, complete answer to the query in a single paragraph. Format: "**{Primary keyword}** is a {category} that {function} for {audience}." |
| **Featured snippet (list)** | H2 = the listicle question. Body = clean `<ol>` with 5–10 short items. |
| **Featured snippet (table)** | H2 = the comparison question. Body = an HTML `<table>` with `<th>` headers. |
| **People Also Ask** | FAQ section with `FAQPage` schema. Use exact question wording from the PAA box. |
| **AI Overview citation** | First-paragraph definition + table + last-updated date + a stat with a source link |
| **Video result** | YouTube embed with descriptive title and a transcript on the page |
| **Image pack** | Original screenshots/diagrams with descriptive alt text and structured filenames |
| **Map Pack** | `LocalBusiness` schema + city/state in H1 and H2s |

## Step 7 — Optimize for AI search (GEO)

ChatGPT, Perplexity, Gemini, and Google AI Overviews now route significant traffic. Apply these to every page:

1. **Answer-first paragraph** in the first 100 words.
2. **Citable statistics** with source links.
3. **Clean H2 hierarchy** that maps to question patterns.
4. **HTML tables and lists** (AI parses them; it cannot parse screenshots).
5. **FAQ section** with `FAQPage` schema.
6. **Updated date** visible on the page (year + quarter).
7. **Author attribution** for blog posts (with `Person` schema).
8. **Brand mentions** in body — AI engines weight brand co-occurrence.

## Step 8 — Internal linking density

For ranking authority to flow into the new page:
- **Inbound**: At least 5 other pages on the site link to this page within their first 1,000 words.
- **Outbound**: This page links to 8–12 other pages on the site (using `sitemap.md`'s linking map).
- **Anchor text variation**: 60% exact-match, 30% partial-match, 10% generic ("learn more", "read here"). Pure exact-match overuse triggers spam penalties.

Wire these the moment the page ships — don't defer.

## Step 9 — Content depth thresholds

| Page type | Word count | Rationale |
|---|---|---|
| Homepage | 800–1,500 (mostly headings + CTA + short blocks) | Conversion-first, not content-first |
| Module page | 1,500–2,500 | Enough depth to win the SEO-keyword landing variants |
| SEO-keyword landing | 2,000–3,500 | Has to outrank existing competitors with deeper content |
| Pricing | 600–1,200 | Don't bury the price |
| Blog post (informational) | 1,500–3,500 | Depth wins for "how to X" queries |
| Blog post (commercial) | 2,500–4,000 | Has to beat ranking competitors |
| Glossary term | 600–1,000 | Definition + 3–4 H2 sections |
| Software review | 2,000–3,500 | Reader expects depth on a review |
| Alternatives | 2,500–4,000 | Lists 10+ alternatives, each with detail |
| Best-of list | 3,000–5,000 | One H2 per ranked tool |
| Case study | 1,200–2,200 | Narrative arc — don't pad |
| Tool page | 600–1,000 | The tool itself is the value; copy supports |
| Vertical landing | 1,500–2,500 | Persona-specific depth |

Going significantly below these counts handicaps the page for ranking. Going significantly above (5,000+) hurts engagement.

## Step 10 — The "boring but essential" technical pass

Run before declaring done:

- ✅ No broken internal links (check every `<a href>` resolves)
- ✅ No orphan pages (this page is linked from at least 1 other page)
- ✅ No duplicate title tags (run a site-wide grep)
- ✅ No duplicate meta descriptions
- ✅ No duplicate H1s
- ✅ Image file size optimized (WebP at quality 80, < 200KB per image)
- ✅ LCP element preloaded if it's an image (`<link rel="preload" as="image" ...>` in `<head>`)
- ✅ Critical CSS inlined (Astro handles this by default)
- ✅ `robots` meta is `index, follow` (no accidental noindex)
- ✅ `hreflang` tags if a localized version exists

## Step 11 — Post-publish monitoring (the next-week todo)

After shipping the page, instrument it:

1. **Submit to Google Search Console** via URL Inspection → Request indexing
2. **Add to `sitemap-0.xml`** (Astro sitemap integration handles automatically on next build)
3. **Wait 7 days**, then check Search Console for impressions
4. **Check the SERP weekly** for ranking position
5. **If page is at position 5–15 after 4 weeks**, optimize: add more entity coverage, more internal links, refresh the date, expand thin sections
6. **If page is below position 30 after 8 weeks**, the intent or keyword choice was wrong — rewrite for a different keyword

## Cheat sheet — keyword density

There is no magic number for keyword density. Modern Google uses **semantic search** + **entity matching**, not raw frequency. The rule:

- Use the primary keyword **once per major section** (H1, first paragraph, ~2x in body, 1x conclusion).
- Use **semantic variations** the rest of the time (synonyms, related terms, partial matches).
- Never repeat the exact keyword more than ~1% of total word count.

If your H1 is "Local SEO Software" and you used "local SEO software" 25 times in 2,000 words, that's 1.25% — borderline. Cut to 1.0%.

## Cheat sheet — title tag patterns that win

Pattern 1: `{Primary Keyword} — {Outcome Promise} | theStacc`
→ `Local SEO Software — Dominate the Map Pack in 90 Days | theStacc`

Pattern 2: `{Outcome} with {Primary Keyword} | theStacc`
→ `Win the Map Pack with Local SEO Automation | theStacc`

Pattern 3 (best-of / list): `{N} Best {Primary Keyword} for {Year} (Tested & Ranked) | theStacc`
→ `12 Best Local SEO Tools for 2026 (Tested & Ranked) | theStacc`

Pattern 4 (review): `{Tool} Review ({Year}): {Verdict} | theStacc`
→ `Ahrefs Review (2026): Worth $129/Month? | theStacc`

Pattern 5 (alternative): `{N} Best Alternatives to {Tool} (Tested) | theStacc`
→ `12 Best Alternatives to Ahrefs (Tested & Compared) | theStacc`

Pick the pattern that matches the page type. Tune the H1 in a similar shape but optimized for click-through (the H1 doesn't have to include the brand name).
